HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body consits of parent elements and child elements
  • The sections we created are part of the body

CSS

  • CSS is presenation component of a web app
  • CSS allows for a better interface with users
  • CSS stands for cascading style sheets
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • DRY stands for do not repeat yourself, and it is a principle that aids in reducing the overall code size and complexity
  • "text-align" is a text alignment positioning property
  • A link element creates the connection to the external file

Git

  • You have to use git commands to interact with git hub
  • Git check -b allows you to create a new feature
  • Git status identifies which branch you are on
  • Git add -A allows you to stage files after saving changes
  • Git commit -m allows you to make commits before pushin
  • Git origin main allows you to pull the latest version of your file from git hub
  • Git push allows you to practically submit what you did so it can be integrated in the main branch

JavaScript

  • JavaScript is the behavioral component that allows useers to interact with the webpages
  • JavaScript is also deemed the "logic" which facilitates interaction
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript files are read from top to bottom. This is known as control flow
  • The array must be defined before the variable when randomized
  • There are 2 requirements for a Javascript fucntion. 1) define a function, 2) call a function.
